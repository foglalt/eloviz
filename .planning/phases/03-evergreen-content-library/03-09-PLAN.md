# Phase 03 Plan 09: Complete Technical SEO and Discoverability

## Objective

Make the published catalogue crawlable, canonical, shareable, and semantically understandable while ensuring raw PDFs do not compete with their descriptive study pages.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-08-SUMMARY.md
- @.planning/research/2026-07-22-evergreen-rebuild-and-bible-v2-feasibility.md

## Tasks

1. Audit metadata for `/`, indexes, topic pages, PDF study pages, and video pages; ensure unique Hungarian titles/descriptions, canonical URLs, Open Graph, and share images.
2. Generate a database-backed `sitemap.xml` containing only canonical published HTML routes with meaningful `lastModified` values.
3. Add `robots.txt`, exclude admin and non-public areas, and verify drafts/staged documents are neither linked nor indexable.
4. Decide and implement the raw-PDF policy. Prefer the descriptive HTML study page as canonical and prevent raw object URLs from becoming competing thin search results where headers/storage capabilities permit.
5. Add page-visible breadcrumbs plus valid `BreadcrumbList`; audit `Article`/`DigitalDocument`, `VideoObject`, `WebSite`, and collection markup for truthfulness.
6. Add semantic landmarks, heading order, descriptive link text, image alternatives, confirmed-reference labels, and internal links exposing topic-study-video relationships.
7. Document Search Console property verification, sitemap submission, URL inspection, raw-PDF checks, and rich-result validation.

## Verification

- Inspect rendered HTML and headers for metadata, canonical, robots directives, structured data, and server-rendered content.
- Validate representative JSON-LD and parse the sitemap.
- Confirm `/admin`, drafts, staged PDFs, candidate references, and error pages are not accidentally indexable.
- Confirm raw PDF URLs follow the chosen indexing policy and public downloads still work.
- Run Lighthouse or equivalent on representative mobile pages.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Every published resource has unique, accurate metadata and one canonical HTML page.
- Raw PDFs remain usable without undermining the dedicated study pages.
- Sitemap, robots, and structured data match publication state.
- Search-engine discovery does not depend on client-only rendering.

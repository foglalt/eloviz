# Phase 03 Plan 05: Build PDF Ingestion and Reference Review

## Objective

Create the secure, versioned PDF upload pipeline that extracts candidate Scripture references and lets an editor finalize an accurate public reference list.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-04-SUMMARY.md

## Tasks

1. Select and document object storage, upload-size limit, retention policy, signed-upload mechanism, and the security boundary between staged and published PDFs.
2. Implement authenticated PDF-only upload with signature/MIME/size validation, sanitized display filename, SHA-256 checksum, immutable revision numbering, and storage cleanup for failed registrations.
3. Extract native PDF text per page and record extraction state/error. Treat image-only or insufficient-text files as “manual references required”; do not add OCR yet.
4. Build a deterministic Hungarian Bible-reference detector using a curated book-name/abbreviation dictionary and tested patterns for common colon/comma, list, range, and cross-chapter forms.
5. Store every candidate with its raw match, normalized canonical range, page number when available, short context snippet, detector version, and initial review state.
6. Build the review/finalization service: accept, correct, reject, manually add, and reorder references; require an explicit “review complete” acknowledgement even for an empty list.
7. Finalize transactionally so the new document revision and confirmed reference set become public together; preserve the prior published revision if extraction, review, or finalization fails.
8. Add fixtures for text PDFs, multi-column/layout-heavy PDFs, malformed references, duplicate/overlapping ranges, Hungarian abbreviations, and scanned PDFs.

## Verification

- Upload valid PDF, wrong MIME, renamed non-PDF, oversized, duplicate, corrupt, native-text, and scanned fixtures.
- Render representative PDFs to images and visually confirm that page numbering/context evidence corresponds to the source pages.
- Compare detector output against a hand-verified expected list for Hungarian reference fixtures.
- Confirm no candidate becomes publicly visible before editor finalization.
- Replace a published PDF, abort review, and confirm the old revision/reference set remains live.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Admin-only PDF uploads are validated, versioned, and stored outside the application filesystem.
- Native-text PDFs produce auditable reference suggestions.
- Scanned PDFs degrade to a clear manual workflow rather than false confidence.
- The editor controls the final reference list.
- Publication switches the PDF and confirmed references atomically.

# Phase 03 Plan 10: Migrate Routes and Verify the Rebuild

## Objective

Cut over to the evergreen site without silent content loss, preserve useful inbound routes, and collect launch-quality verification across PDF ingestion, reference review, and public discovery.

## Context

- @.planning/PROJECT.md
- @.planning/REQUIREMENTS.md
- @.planning/ROADMAP.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-09-SUMMARY.md

## Decision Gate

Before destructive route cleanup, choose whether to preserve the Easter experience under `/husvet`, retain selected legacy routes, or retire it with explicit permanent redirects. No old content is deleted without this choice and a backup.

## Tasks

1. Back up production database content and object-storage inventory; record deployed schema/migration and detector revisions.
2. Execute the approved route migration and add permanent redirects without chains.
3. Remove obsolete code only after content parity and route behavior are proven; retain import scripts, PDF provenance, checksum, and attribution records.
4. Run an end-to-end editorial test from study creation through PDF upload, extraction, candidate correction, finalization, publication, public display, replacement, and unpublish.
5. Test native-text, scanned, corrupt, missing-object, landscape, and large allowed PDF cases plus duplicate/overlapping references.
6. Test responsive layout, keyboard, screen-reader landmarks, contrast, reduced motion, 200% zoom, slow network, database/storage failure, removed video, draft access, and not-found behavior.
7. Check metadata, structured data, sitemap, robots, raw-PDF headers, canonical URLs, redirects, and page status codes in a production-like build.
8. Resolve launch blockers, run phase verification/health, create the summary, and update state/roadmap/requirements accurately.

## Verification

- `npm run lint`
- `npm run build`
- Production-server smoke test for all public/admin routes.
- Automated/manual accessibility checks on home, topic, study, video, login, PDF review, and editor pages.
- Hand-verified PDF reference extraction fixture report.
- Redirect/status-code map for every replaced legacy URL.
- Database and object-storage backup/restore or disposable-environment rehearsal.
- `./scripts/gtd.ps1 verify -Phase 03`
- `./scripts/gtd.ps1 health`

## Success Criteria

- The evergreen library is primary and all requested V1 features work together.
- Editors can reliably turn a PDF into a published study with a verified verse list.
- Confirmed mappings are ready for Phase 04 verse-to-study queries.
- Existing valuable content is migrated or deliberately preserved.
- No launch-blocking accessibility, security, data-integrity, storage, or build issue remains.
- Phase 03 has complete summaries, verification evidence, and accurate state.

# Phase 03 Plan 08: Build the Editorial Admin

## Objective

Turn `/admin` into a simple, secure content workbench for topics, PDF studies, videos, publication state, extracted-reference review, topic membership, and study/video relationships.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-07-SUMMARY.md
- @src/app/admin
- @src/lib/admin-auth.ts

## Tasks

1. Harden the existing one-password flow with a separate `ADMIN_SESSION_SECRET`, secure cookie settings, session expiry, constant-time checks, basic login throttling, and consistent authorization in every action and admin page.
2. Replace the quiz-only dashboard with clear navigation and list views for topics, studies, and videos, showing publication/review state and last update.
3. Implement create/edit forms using Server Actions and `useActionState`, with field-level Hungarian errors and explicit save/publish/unpublish actions.
4. Integrate the PDF revision workflow: upload progress, validation/extraction status, PDF preview/open action, candidate list with page/context evidence, manual reference entry, and finalization acknowledgement.
5. Make it impossible to publish a study without a valid finalized PDF revision and completed reference review; allow a deliberately confirmed empty list.
6. Add topic assignment and study/video relation editors, including ordering and optional relation notes.
7. Validate slug uniqueness, block silent published-slug changes, validate YouTube URLs, and show previews of final public URLs.
8. Make unpublish the safe default. Require explicit confirmation for deletion and preserve published document history according to retention policy.
9. Revalidate every affected public/admin path, sitemap, relation, raw-file policy, and future verse cache key after a successful finalization.

## Verification

- Attempt every mutation signed out, with expired/tampered cookie, and authenticated.
- Exercise create/edit/upload/extract/review/finalize/publish/unpublish for native-text and scanned PDFs.
- Test duplicate slugs, bad YouTube URLs, invalid PDF, broken relations, stale edits, and interrupted extraction.
- Verify keyboard use, field labels, errors, focus management, and narrow-screen review UI.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- A trusted non-technical editor can manage all requested content without editing code or JSON.
- The PDF/reference review state is always clear.
- All automated detections remain editable suggestions until finalization.
- Study publication is blocked until its PDF and reference review are internally consistent.
- Successful edits appear publicly without a redeploy.

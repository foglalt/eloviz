# Phase 03 Plan 07: Build Video Pages and Related Content

## Objective

Create standalone Hungarian description pages for recommended YouTube videos, with safe playback/link behavior, topic context, and explicitly related PDF studies.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-06-SUMMARY.md

## Tasks

1. Build `/videok` as a server-rendered index of published recommendations with editorial descriptions and topic context.
2. Build `/videok/[slug]` with a stable watch-page layout, unique Hungarian description, channel/source attribution, thumbnail, outbound YouTube link, topic links, and related studies.
3. Accept only recognized YouTube URL formats, normalize to a video ID, escape all editor-controlled values, and handle removed, private, age-restricted, or non-embeddable videos gracefully.
4. Implement the chosen privacy-conscious playback treatment with no autoplay. Prefer privacy-enhanced embed URLs and document any consent tradeoff.
5. Generate unique metadata, canonical and Open Graph values, breadcrumbs, and `VideoObject` only when the required visible facts and player data are present.
6. Make study/video relations visible in both directions with optional relation notes and deterministic ordering.

## Verification

- Test supported/invalid YouTube URL forms and embedding failures.
- Confirm no autoplay and inspect network/privacy behavior.
- Confirm related-study symmetry, draft exclusion, and links to PDF study pages.
- Validate visible facts against metadata/JSON-LD.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Every published video recommendation has substantial unique Hungarian copy on its own page.
- Visitors can reach the video even when embedding fails.
- Related PDF studies are accurate and visible in both directions.
- Structured video data is emitted only when valid and truthful.

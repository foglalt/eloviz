# Phase 03 Plan 02: Migrate and Seed Evergreen Content

## Objective

Move useful existing study metadata out of source-code constants into the new content model, repair encoding, and create an idempotent initial catalogue without fabricating PDF uploads or removing legacy routes.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-01-SUMMARY.md
- @src/app/(husvet)/_content/studies-content.ts

## Tasks

1. Inventory the five existing study entries, their sections, Scripture references, completion state, and any useful SEO copy.
2. Define a small initial topic taxonomy with meaningful Hungarian descriptions; avoid making every study its own topic.
3. Convert current titles, summaries, topics, and known Scripture references into durable records, correcting mojibake from verified source text. Preserve longer legacy text only as migration source/notes, not as a substitute uploaded PDF.
4. Implement an idempotent seed/import command that upserts known records and relationships without overwriting later editorial changes unless explicitly requested.
5. Seed a minimal set of representative videos only when real YouTube URLs and descriptions are available; otherwise leave video fixtures clearly non-production and keep the public catalogue truthful.
6. Produce a migration report mapping old study IDs/titles to new slugs, known canonical references, and PDF availability. Records without a real source PDF remain draft or explicitly legacy until supplied.

## Verification

- Run the import twice and confirm record counts and editorial fields remain stable.
- Compare every migrated title, paragraph, list, and Scripture reference with the UTF-8 source.
- Confirm incomplete placeholder studies are drafts or visibly marked, not presented as finished material.
- Query topics, studies, and relations directly and review the migration report.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Existing valuable material is stored in Postgres and no longer depends on a page-specific TypeScript constant.
- Migrated Hungarian text renders without encoding corruption.
- The seed is safe to rerun.
- Every imported study has at least one topic and normalized confirmed Scripture references where appropriate, without claiming automated PDF verification occurred.
- No study is published as a PDF resource unless an actual validated PDF revision exists.
- No legacy route or content is deleted in this plan.

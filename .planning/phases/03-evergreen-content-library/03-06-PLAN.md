# Phase 03 Plan 06: Build SEO-First Study Pages

## Objective

Give every published PDF study a high-quality standalone HTML page with meaningful Hungarian description, confirmed Scripture references, topic context, PDF access, and related videos.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-05-SUMMARY.md

## Tasks

1. Build `/tanulmanyok` as a server-rendered index of published studies with summary, topic context, PDF availability, and stable ordering.
2. Build `/tanulmanyok/[slug]` with title, unique description, confirmed Scripture references, topic links, current PDF view/download action, document update information where useful, and related videos.
3. Provide an accessible PDF treatment with a clear download/open fallback. Do not make an embedded browser viewer the only way to obtain the material.
4. Display confirmed verse ranges as structured links ready to target Phase 04 reader URLs; until the reader exists, render useful labels without dead links.
5. Generate unique title, description, canonical, Open Graph data, and truthful `Article`/`DigitalDocument` plus `BreadcrumbList` JSON-LD from page-visible facts.
6. Return not found for draft/unknown slugs and prevent publication when there is no finalized PDF revision or reference-review acknowledgement.
7. Serve raw PDFs from the storage boundary without application cookies and with the indexing/canonical policy finalized in 03-09.

## Verification

- Review native-text, scanned, long, landscape, and multi-page PDF examples on mobile and desktop.
- Confirm the public page shows only the selected published document and confirmed references.
- Confirm a draft document revision and candidates cannot leak through HTML, download URLs, metadata, or caches.
- Validate metadata/JSON-LD against visible content.
- Test keyboard access, download fallback, missing-object behavior, readable line length, and mobile overflow.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Every published study has a descriptive, indexable HTML page and a valid PDF revision.
- Confirmed references are visible and structurally ready for reader backlinks.
- Visitors can open or download the PDF reliably.
- Staged documents and unreviewed candidates remain private.
- Metadata accurately describes the study page rather than pretending the HTML contains the full PDF text.

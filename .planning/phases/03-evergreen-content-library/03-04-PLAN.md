# Phase 03 Plan 04: Build Homepage and Topic Discovery

## Objective

Deliver the new public root, complete topic index, and individual topic pages using real published content from the database.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-03-SUMMARY.md

## Tasks

1. Build `/` as an async Server Component using the agreed hero, selected topics, one related study/video feature, and restrained latest/featured resource lists.
2. Build `/temak` with all published topics, concise descriptions, useful resource counts, and stable ordering.
3. Build `/temak/[slug]` with the full topic description and separate study/video sections that make content type obvious without duplicating page chrome.
4. Add loading, empty, database-unavailable, and not-found states that remain truthful and helpful in Hungarian.
5. Add internal navigation between the homepage, topic index, topic detail, studies, and videos.
6. Add page-specific metadata and breadcrumb data hooks without duplicating the full SEO work reserved for 03-08.

## Verification

- Confirm drafts never appear in home/topic queries or counts.
- Confirm a missing or draft topic returns the intended not-found behavior.
- Review topics with only studies, only videos, both, and neither.
- Check semantic heading order and link names by keyboard and screen-reader inspection.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- The root clearly communicates the new purpose and directs visitors into topics.
- Every published topic has a readable dedicated page.
- Topic pages show associated studies and videos accurately.
- Empty and error states do not expose internal details or invent content.
- The composition matches the Living Water thesis on mobile and desktop.

# Phase 03 Plan 03: Create the Living Water Design System and Shell

## Objective

Replace the Easter-specific visual shell with a distinctive, accessible Élő Víz brand system that supports both editorial public pages and a restrained utility admin.

## Context

- @.planning/PROJECT.md
- @.planning/STATE.md
- @.planning/phases/03-evergreen-content-library/03-CONTEXT.md
- @.planning/phases/03-evergreen-content-library/03-02-SUMMARY.md

## Visual Thesis

Deep spring water meeting warm limestone: contemplative, luminous, tactile, and quietly alive.

## Content Plan

1. Hero with brand, Revelation 22:17 invitation, primary topic CTA, and one dominant natural-water image.
2. Editorial topic current with typography and dividers instead of a generic card grid.
3. A featured study/video pairing that demonstrates the catalogue’s related-content model.
4. A final invitation returning to the living-water promise.

## Interaction Thesis

- Short hero entrance suggesting water coming into light.
- One restrained scroll-depth or masked-light effect.
- Consistent underline/current and media-reveal transitions, all with reduced-motion alternatives.

## Tasks

1. Establish colour, typography, spacing, radius, focus, elevation, reading-width, and motion tokens; use two font families at most and verify Hungarian glyph coverage.
2. Source or generate one strong natural-water visual with a calm text area and responsive crops; record usage rights and optimization requirements.
3. Build the public site header, mobile navigation, footer, skip link, page container, breadcrumbs, reading prose, media rows, empty states, and error/not-found treatments.
4. Add the Revelation 22:17 working quote and translation note, then verify exact wording/attribution before marking the plan complete.
5. Keep public pages primarily cardless; use contained surfaces only when the containment has semantic or interactive purpose.
6. Create separate admin shell primitives with dense, clear operational labels and no marketing hero.
7. Verify contrast, focus visibility, touch targets, zoom, reduced motion, and first-viewport behavior on common mobile and desktop sizes.

## Verification

- Visual review at 360, 390, 768, 1280, and 1440 CSS pixels.
- Keyboard-only navigation and focus-order review.
- `prefers-reduced-motion` review.
- Automated accessibility smoke check on the shell where tooling is available.
- `npm run lint`
- `npm run build`
- `./scripts/gtd.ps1 verify -Phase 03`

## Success Criteria

- Élő Víz is unmistakable in the first viewport.
- The site has one strong water-led visual anchor and a restrained reusable design system.
- Editorial pages are comfortable for long Hungarian reading.
- The admin looks and reads like an operational workspace.
- The shell remains usable with keyboard, zoom, reduced motion, and narrow viewports.

# Requirements

## Completed Baseline

- [x] **REQ-01**: Working Next.js 16 / React 19 project with lint and production build checks.
- [x] **REQ-02**: Structured project memory and verification workflow.
- [x] **REQ-03**: Neon-backed content precedent and password-protected admin session.
- [x] **REQ-04**: Existing Hungarian Easter experience, quiz, and five study entries preserved as migration sources.

## Phase 03 — Evergreen Content Library V1

- [x] **REQ-15**: Replace the root experience with a Hungarian evergreen Élő Víz content library.
- [x] **REQ-16**: Let visitors browse topics, each with a title, description, stable slug, and dedicated public page.
- [x] **REQ-17**: Let visitors browse individual Bible-study materials on dedicated descriptive pages and open/download the attached PDF.
- [x] **REQ-18**: Let visitors browse recommended YouTube videos on dedicated descriptive pages.
- [x] **REQ-19**: Allow studies and videos to belong to multiple topics and to be explicitly marked as related to one another.
- [x] **REQ-20**: Provide a simple authenticated admin for creating, editing, publishing, unpublishing, ordering, and safely deleting topics, studies, videos, and their relations.
- [x] **REQ-21**: Redesign the public site around an accessible, restrained “living water” visual identity with the Revelation 22:17 invitation.
- [x] **REQ-22**: Give every published topic, study, and video unique metadata, canonical URLs, structured data where valid, and sitemap inclusion.
- [x] **REQ-23**: Import useful existing study content into the new model and preserve or redirect old URLs without silent content loss.
- [x] **REQ-24**: Exclude drafts from public routes, metadata, structured data, and sitemaps.
- [x] **REQ-25**: Verify mobile usability, keyboard access, reduced motion, contrast, missing-content behavior, lint, build, and memory integrity before launch.
- [x] **REQ-27**: Store study PDFs in object storage as versioned documents with validated file type, size, checksum, and publication state.
- [x] **REQ-28**: Extract text and candidate Bible references from a newly uploaded or replaced PDF, retaining page/context evidence where available.
- [x] **REQ-29**: Let an authenticated editor confirm, correct, add, reject, and reorder detected references before finalizing the study revision.
- [x] **REQ-30**: Publish only confirmed Scripture references and model them against canonical verses so a later reader can query which studies contain a verse.

## V2 Feasibility Requirement

- [x] **REQ-26**: Research and document the feasibility, licensing risks, data model, and staged scope for multiple Bible translations plus Greek/Hebrew concordances.
- [ ] **REQ-31**: In Phase 04, deliver one simple licensed/open Hungarian Bible reader with per-verse links to matching published studies before adding concordances.

## Explicitly Out of Scope for V1

- Full Bible reader or concordance implementation.
- Word-level alignment between Hungarian translations and Greek/Hebrew source texts.
- Multi-user roles, account recovery, or a general-purpose CMS.
- General-purpose media management beyond admin-only study PDF uploads.
- Automatic theological summarization or AI-generated study content.
